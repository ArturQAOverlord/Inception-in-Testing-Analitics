№A152. При формировании квитанции в записи мониторинга операции перевода между картами "Платежной системы" - номер транзакции в квитанции не соответствует таковой в записи мониторинга и заново генерируется при каждом формировании квитанции.
Окружение: 
REDMI NOTE 13 / ANDROID 14/ 1.0.37(135)
Предусловие:
1) Пользователь зарегистрирован, авторизован, идентифицирован
2) Осуществлена операция перевода между картами "Платежная система" "Банка" в приложении
3) Открыт таб мониторинга, таб “Мои операции”
Воспроизведение:
Шаг 1. Тапнуть по записи из предусловия в списке таба “Мои операции”
Шаг 2. Зафиксировать номер транзакции в записи операции
Шаг 3. Тапнуть по кнопке просмотра квитанции операции(“Чек”)
Шаг 4. Зафиксировать номер транзакции в квитанции операции
Шаг 5. Тапнуть по апбаттону приложения
Шаг 6. Повторно тапнуть по кнопке просмотра квитанции операции(“Чек”)
Шаг 7. Зафиксировать номер транзакции в квитанции операции
Фактический результат: 
Зафиксированные номера транзакций отличаются, при формировании квитанции операции - номер транзакции в квитанции генерируется заново, пока номер транзакции в записи мониторинга статичен.
Ожидаемый результат:
Зафиксированные номера транзакций идентичны, при формировании квитанции операции - номер транзакции в квитанции статичен, номер транзакции в записи мониторинга статичен.
Серьёзность: Критичный
